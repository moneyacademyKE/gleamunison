Gleamunison Standard Library

Core Modules

gleamunison/identity

Content-addressed identity primitives. Opaque Hash type, SHA256 hashing, definition references, local variables (de Bruijn indices), and 40+ genesis builtins.

hash_bytes/1 — Compute SHA256 hash from bytes
hash_equal/2 — Compare two hashes for equality
hash_to_debug_string/1 — Hex string for display
builtin_int_add/0 — Genesis builtin: integer addition
builtin_http_get/0 — Genesis builtin: HTTP GET

gleamunison/ast

Core AST types: Type, Term (15 variants), Pattern, Case (with optional Guard), Definition, Unit. All identity is structural — no names in the AST.

Term — Int, Float, Text, List, Lambda, Apply, Let, Match, Do, Handle, Construct, Hole NEW, Use NEW
Guard NEW — GuardTerm wrapping a BEAM-guard-safe Term

gleamunison/types

Type checking infrastructure: TypeCache, InferenceError (with LinearityViolation NEW), HandlerError, validate_handler.

validate_handler/3 — Compile-time handler validation against ability declarations
check_linearity/2 NEW — Validate continuation variables used exactly once

gleamunison/compile

Erlang source generation from AST, in-memory compilation via erl_scan/erl_parse, and module loading. Supports Guard NEW, Hole NEW, and Use NEW variants.

compile_definition/3 — Compile a Definition to BEAM binary
module_name_for/1 — Generate m_ prefixed module name from hash

gleamunison/codebase

Content-addressed definition store. Hash-of-definition computation for all 15 Term variants, insertion with hash verification.

hash_of_definition/1 — Compute SHA256 hash from AST + inferred type
insert/2 — Insert verified definitions into codebase

gleamunison/effects

Algebraic effects runtime: HandlerFrame, OpHandler, RuntimeConfig.

run/2 — Execute computation with ambient handler stack
