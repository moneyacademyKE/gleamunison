# ADR-0057: v3.4.0 Rich Hickey Gap Analysis and Evolution Path

## Context
As of v3.4.0, the `gleamunison` runtime is fully certified, passing all 2270 playbook conformance checks. To direct the future evolution of the project, we needed to perform a comprehensive Rich Hickey Gap Analysis comparing `gleamunison` (v3.4.0) with the latest features of Unison (v1.1.0/1.3.0, including AI MCP servers, git-style diffs, and dependents queries) and Gleam (v1.17.0, including native escript compilation and todo constants).

## Decision
We evaluated six potential feature additions using a weighted Power/Complexity matrix:
1. **Code Graph Querying (`dependents` / `dependencies` commands)** (Utility: 8, Complexity: 3) -> **Priority 1 (Accepted)**
2. **AI MCP Server Integration** (Utility: 9, Complexity: 5) -> **Priority 2 (Accepted)**
3. **Context-Aware Diagnostics spelling/type hints** (Utility: 6, Complexity: 4) -> **Priority 3 (Accepted)**
4. **Git-style Patch/Propagation lists (`diff.update`)** (Utility: 10, Complexity: 9) -> **Deferred**
5. **Native Escript Export** (Utility: 4, Complexity: 5) -> **Rejected (Custom build script is superior)**
6. **Todo in constants** (Utility: 2, Complexity: 3) -> **Rejected (Minimal developer utility)**

We will prioritize implementing Code Graph Queries and AI MCP Server Integration in subsequent iterations.

## Consequences
- The project maintains its core design constraint of keeping files <250 LOC by avoiding the high-complexity patch engine and native escript changes.
- Observability and pair-programming ergonomics will be significantly enhanced through MCP integration and transitive graph lookups.
